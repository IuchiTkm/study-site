Git
コミット
変更内容をリポジトリに記録するスナップショット。
ブランチ
コミット履歴の分岐。機能開発や修正を独立して進めるために使う。
リポジトリ
変更履歴を管理するためのデータの保管場所。
Linux
パーミッション
ファイルやディレクトリに対する読み・書き・実行の権限設定。
プロセス
実行中のプログラムの単位。PID（プロセスID）で識別される。
Git
コマンド 説明
git add <file> 変更をステージングエリアに追加する
git clone <url> リモートリポジトリをローカルにコピーする
git commit -m "<msg>" ステージした変更をコミットする
git init 新しいリポジトリを初期化する
git pull リモートの変更を取得してマージする
git push ローカルのコミットをリモートに送信する
git status 作業ツリーの状態を確認する
Linux
コマンド 説明
cd <dir> ディレクトリを移動する
chmod <mode> <file> ファイルのパーミッションを変更する
grep <pattern> <file> ファイル内からパターンを検索する
ls -la ファイル一覧を詳細表示（隠しファイル含む）
pwd 現在のディレクトリパスを表示する
